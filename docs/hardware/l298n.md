# L298N — GPIO H-Bridge

The L298N is an inexpensive dual H-bridge chip widely available as a breakout board. It is the most common motor driver for hobbyist wheeled robots.

```yaml
hardware:
  type: "l298n"
  options:
    forward_pins: [17]
    backward_pins: [18]
    left_pins: [22]
    right_pins: [23]
    drive_seconds: 0.35
    turn_seconds: 0.20
```

---

## How it works

The adapter drives the motor direction by pulsing GPIO output pins HIGH for a fixed duration, then setting them LOW again. This is a "bang-bang" (open-loop) approach — there is no speed feedback.

Each command (`forward`, `backward`, `left`, `right`) maps to a set of pins that are pulsed for the configured number of seconds.

---

## Wiring

The L298N has two logic inputs per motor channel (IN1/IN2 for motor A, IN3/IN4 for motor B). Connect the input pins to Raspberry Pi GPIO pins.

```
Raspberry Pi GPIO 17 ──► L298N IN1  (motor A forward)
Raspberry Pi GPIO 18 ──► L298N IN2  (motor A backward)
Raspberry Pi GPIO 22 ──► L298N IN3  (motor B left)
Raspberry Pi GPIO 23 ──► L298N IN4  (motor B right)
Raspberry Pi GND     ──► L298N GND
```

> **Enable pins**: Bridge the EN_A and EN_B jumpers on the L298N board OR connect them to 3.3 V to keep both channels always enabled. If left floating the motors will not move.

The L298N also requires its own power supply (6–35 V). Connect battery positive/negative to the `+12V` and `GND` terminals of the L298N. The onboard 5 V regulator can power the Raspberry Pi if your supply is ≥7 V (bridge the 5 V jumper).

---

## Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `forward_pins` | list[int] | `[]` | BCM GPIO pins to pulse for forward movement |
| `backward_pins` | list[int] | `[]` | BCM GPIO pins to pulse for backward movement |
| `left_pins` | list[int] | `[]` | BCM GPIO pins to pulse for left turn |
| `right_pins` | list[int] | `[]` | BCM GPIO pins to pulse for right turn |
| `drive_seconds` | float | `0.35` | Duration of each drive pulse in seconds |
| `turn_seconds` | float | `0.20` | Duration of each turn pulse in seconds |

You can include multiple pins per direction to control both motors simultaneously:

```yaml
hardware:
  type: "l298n"
  options:
    forward_pins: [17, 27]   # two pins → two motor channels forward
    backward_pins: [18, 22]
    left_pins: [17, 22]      # left wheel forward, right wheel backward
    right_pins: [27, 18]
```

---

## Tuning drive and turn times

- If the robot overshoots on turns, reduce `turn_seconds`.
- If straight movement is too short, increase `drive_seconds`.
- A typical differential-drive robot needs `drive_seconds` between 0.25 and 0.5 s and `turn_seconds` between 0.15 and 0.25 s.

---

## Dependencies

```bash
sudo apt install python3-rpi.gpio
sudo usermod -aG gpio $USER   # log out and in after this
```

If Raspberry Pi OS proposes removing `python3-rpi-lgpio`, that is normal. The packages conflict because they provide overlapping GPIO compatibility layers.

The RPi.GPIO library is imported dynamically at startup. If it is not installed the adapter runs in **simulation mode** — commands are logged but no GPIO is driven. This is useful for testing on a non-Pi machine.
